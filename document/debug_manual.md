# デバッグモード・マニュアル

このドキュメントでは、開発者向けに実装されたデバッグモードの使い方を説明します。
デバッグモードを使用することで、通常は見えないCPUの手札情報を確認したり、挙動の検証を行うことが可能です。

## 使用方法

### 1. ブラウザコンソールの起動
ゲーム画面（ブラウザ）上で、以下のキーを押して開発者コンソールを開いてください。
*   `F12` キー
*   または `Ctrl + Shift + I`

### 2. コマンドの入力
コンソール（Console）タブに、以下のコマンドを入力して実行（Enter）してください。

#### 手札の一覧表示（推奨）
現在の全プレイヤーの手札をコンソールに出力します。

```javascript
window.debug.showHands()
```
または単純に
```javascript
debug.showHands()
```

**出力例:**
```
Shiina (10): ♠3, ♥5, ♦J...
Saki (8): ♣2, ♠A...
You (5): ♦3, ♦4, ♦5...
```

#### デバッグモード切替（トグル）
現在、追加の機能はありませんが、将来的に画面上にデバッグ情報を常時表示するモードの切替用です。

```javascript
window.debug.toggle()
```

## コンソールログの確認方法

ゲーム進行中、カードが出されるたびに以下の形式でログが出力されます。

```
[CharacterName] played: ♠3, ♥3 (2 cards)
```
*   **Suit (スート):**
    *   ♠ (Spade)
    *   ♥ (Heart)
    *   ♦ (Diamond)
    *   ♣ (Club)
*   **Rank (ランク):**
    *   3, 4, ..., 9, 10
    *   J (Jack), Q (Queen), K (King)
    *   A (Ace), 2

これにより、実際にどのカードが出されたかをテキストで正確に追跡可能です。

## トラブルシューティング

*   **`debug` is not defined:**
    *   ページのリロードを行っていない可能性があります。一度リロードしてください。
    *   `game_core.js` が正しく読み込まれているか確認してください。

*   **Game not started:**
    *   ゲーム（対戦）がまだ開始されていません。「対戦開始」ボタンを押してゲーム画面に進んだ後にコマンドを実行してください。
